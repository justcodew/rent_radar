# 使用手册

本文档面向普通用户，说明租房雷达的各项功能怎么用。

## 快速上手

打开网页后，顶部导航栏有这些功能入口：

```
首页 | 搜索 | 案例 | 采集 | AI选房 | 小区测评 | 推荐 | 需求画像 | 收藏
```

## 一、搜索房源 🔍

**入口**：导航栏 →「搜索」

在搜索页可以：
- **关键词搜索**：输入"越秀 两房 电梯"等关键词
- **区域筛选**：勾选"越秀区""天河区"等
- **价格筛选**：设置最低/最高价格
- **评分筛选**：只看好房指数 ≥ 75 的房源
- **排序**：按评分/价格/时间排序

每张房源卡片上会显示：
- 🏷️ **关键标签**：房型 / 面积 / 🚶步梯5楼中层 / 🛗电梯3楼 / 🌿阳台 / 🧭南向 / 🚇农讲所
- ⭐ **好房指数**（满分 100）
- 🏷️ **中介标记**（橙色"疑似中介"标签）
- ✅ **已租出标记**（半透明 + 角标）
- 📕 **来源**（小红书/豆瓣）

点击卡片进入详情页。

## 二、房源详情页 📄

进入详情后有三个 Tab：

### 📄 原帖内容（默认）
- 完整的帖子正文
- 联系方式（如有微信/手机号）
- 原文关键词高亮
- 关键标签区（房型/楼层/电梯/阳台/朝向/地铁）

### ✨ AI 深度洞察
点击「✨ 生成 AI 深度洞察」按钮，AI 会以"老广朋友"口吻分析：
- 小区画像（什么年代建的、住什么人）
- 周边配套（地铁/学校/医院/商场，含步行距离）
- 5 优点 + 5 缺点
- 价格评价（跟区域均价对比）
- 3 条看房/砍价建议
- 推荐结论（值得看/谨慎看/别看）

### 📊 评分雷达
- 8 维度雷达图（发布频率/账号年龄/内容多样性/图片真实性/描述详细度/价格合理性...）
- 好房指数总分 + 等级（精选好房/优质好房/一般/不推荐）
- 风险标签（疑似中介/价格异常低/图片存疑）

## 三、需求案例 📋

**入口**：导航栏 →「案例」

展示典型租房需求案例，例如：
> "越秀地铁站附近 4K 能租到带电梯阳台的方正两房吗？"

页面包含：
- **需求清单**：预算/地铁站/距离/小区环境/房间布局/朝向/周边环境
- **AI 推荐片区**：4 个适合的片区 + 推荐理由
- **现实分析**：结论（有难度但不是不可能）+ 详细分析 + 看房建议
- **匹配房源**：从数据库实时匹配的房源，点击可进详情

点击右上角「🔄 刷新数据」按钮可以：
- 重新匹配最新房源
- AI 基于最新数据重新分析

## 四、采集房源 🕷️

**入口**：导航栏 →「采集」

### 前提条件
1. Chrome 浏览器已开启远程调试（端口 9222）
2. 在 Chrome 窗口里已登录小红书/豆瓣/微博

### 操作步骤
1. 选择平台（小红书/豆瓣/微博）
2. 输入关键词（如"广州越秀两房出租"）
3. 设置最大条数（建议 20-50）
4. 点击「开始采集」
5. 页面显示"🔄 采集中..."（每 3 秒自动刷新状态）
6. 采集完成后显示"✅ 采集完成！N 条数据"
7. 点击「入库+评分」把数据导入数据库

采集过程会自动：
- 下载图片到本地
- 反检测（截图风控感知 + 行为拟人化）
- 断点续爬（中断后可继续）
- 提取结构化标签（价格/户型/楼层/电梯/地铁站等）

## 五、AI 选房 🤖

**入口**：导航栏 →「AI 选房」

用自然语言描述你的需求，AI 帮你找房：

> 例："我在体育西上班，预算 3000，想要两房一厅，近地铁，安静小区，南向，有阳台"

AI 会：
1. 提取结构化需求（预算/区域/户型/通勤/关键词）
2. 推荐 3-5 个适合的小区
3. 从数据库匹配实际房源

## 六、小区测评 🏘️

**入口**：导航栏 →「小区测评」

输入小区名或地名，AI 帮你综合分析：

> 例：输入"公园前"或"农林上路"

AI 以"老广"口吻分析：
- 小区画像（什么类型/住什么人/整体定位）
- 周边配套（地铁/学校/医院/商场）
- 5 优点 + 5 缺点
- 价格评价
- 3 条看房建议
- 推荐结论

### 自定义提示词
页面下方有可折叠的「AI 提示词」编辑器：
- 默认折叠，点击展开
- 可修改 System Prompt（AI 人设）和 User Prompt（分析指令）
- 保存后影响后续分析
- 可随时恢复默认

## 七、推荐 📊

**入口**：导航栏 →「推荐」

基于你的**租房画像**个性化推荐。

使用步骤：
1. 先在「需求画像」创建画像（预算/区域/户型/通勤等）
2. 回到「推荐」页，选择画像
3. 系统按匹配度排序推荐房源

匹配维度：价格(20%) + 通勤(20%) + 区域(15%) + 户型(15%) + 环境(15%) + 关键词(10%)

## 八、需求画像 👤

**入口**：导航栏 →「需求画像」

创建你的租房需求画像：
- **预算范围**：最低 ~ 最高
- **区域偏好**：勾选目标区域
- **户型偏好**：一房/两房/三房
- **通勤需求**：工作地点 + 地铁/公交
- **环境偏好**：安静/近公园/近超市等
- **关键词**：电梯/阳台/民水民电等

可创建多个画像（如"越秀上班版"和"天河上班版"）。

## 九、收藏 ⭐

**入口**：导航栏 →「收藏」

在房源详情页可以收藏到不同分类：
- **待看**：准备去看的
- **看过**：已经看过的
- **不考虑**：排除的
- **已租**：已经租了的

还可以标记反馈：
- 标记中介 / 疑似虚假 / 环境吵 / 环境安静

## 标签说明

| 标签 | 含义 |
|------|------|
| 🏠 两房一厅 | 房型 |
| 📐 60㎡ | 面积 |
| 🚶 步梯5楼，中层 | 步梯 5 楼(中层略累) |
| 🚶 步梯9楼，超高层⚠️ | 步梯 9 楼(慎重!) |
| 🛗 电梯6楼 | 有电梯，6 楼 |
| 🛗 原生电梯3楼 | 原生电梯(非加装) |
| 🔧 加装电梯·中楼层 | 后加装电梯 |
| 🌿 阳台 | 有阳台/露台 |
| 🧭 南向 | 朝向 |
| 🚇 农讲所 | 附近地铁站 |
| ✅ 已租出 | 该房源已租出 |
| 🏷️ 疑似中介 | 发帖人疑似中介/职业二房东 |

## 中介识别说明

系统从 7 个维度判断发帖人是否为中介：
1. 帖子内容关键词(专业租房/多家房源/随时看房等)
2. 昵称特征(含中介/房产/置业/公寓/优选好房等)
3. 发布频率(同一账号发多条)
4. 联系方式复用(微信/手机多帖出现)
5. 模板化标题(emoji 堆砌/固定格式)
6. 营销昵称模式
7. 内容多样性(标题高度雷同)

判定结果：
- 🏷️ 高度疑似中介(3+信号) → 橙色标签
- 🏷️ 疑似中介(2信号) → 橙色标签
- ℹ️ 可能个人(轻度营销)(1信号) → 灰色标签
- 无标记 → 个人房东
